📱 APPLICATION FLOW
═══════════════════════════════════════════════════════════════════════════════

User Journey:
───────────

Step 1: Identity Portal
└─ User logs in or creates account
   • Select language (English/Kinyarwanda)
   • Enter Business ID
   • Verify identity through RDB

Step 2: Two Paths Diverge

   PATH A: Running Businesses (with tax history)
   └─ Eligibility Dashboard
      • System verifies RRA tax records
      • Algorithm calculates eligible loan amount
      • User sees qualification score
      • Clear loan amount cap displayed

   PATH B: Early-Stage Founders (no tax history)
   └─ Narrative Form
      • User describes business idea
      • Explains market opportunity
      • Details loan purpose
      • Shows repayment confidence
      • BRD will assess manually

Step 3: Loan Configuration
└─ Loan Purpose Module
   • Select requested loan amount (slider)
   • Choose loan purpose category
   • Select repayment timeline
   • Review monthly payment estimate
   • Submit application

Step 4: Application Tracking
└─ Real-Time Tracker
   • See current application status
   • Track progress through stages:
     1. Application submitted
     2. Data verification (RRA/RDB)
     3. Eligibility assessment
     4. BRD human review
     5. Final decision
     6. Fund disbursement
   • Monitor estimated timeline
   • View reviewer information

Step 5: Decision & Disbursement
└─ Get loan decision within 24 hours
   • Approval: Funds in 2-3 business days
   • Request more info: Respond in portal
   • Respectful decline: Next steps provided

🎯 KEY DESIGN DECISIONS
═══════════════════════════════════════════════════════════════════════════════

Why This Design?

1. Split-Screen Identity Portal
   • Creates visual hierarchy
   • Hero message builds trust
   • Form clearly secondary
   • Professional appearance
   • Desktop-friendly, mobile-responsive

2. Two Pathways (Tax vs. Narrative)
   • Serves both business types
   • No paperwork required
   • Clear differentiation
   • Equal opportunity
   • Appropriate for context

3. Category Cards for Selection
   • More intuitive than dropdowns
   • Visual icons help non-readers
   • Supports bilingual labeling
   • Better for mobile
   • Reduces cognitive load

4. Interactive Slider for Amount
   • Visual feedback on selection
   • Shows available range
   • More engaging interface
   • Mobile-friendly
   • Real-time calculation

5. Timeline for Status
   • Natural progression display
   • Clear before/after context
   • Shows completion vs. pending
   • Builds user confidence
   • Reduces support inquiries

6. Multi-Section Narrative Form
   • Reduces cognitive load
   • Lets user focus one section
   • Easier to revisit sections
   • Mobile-friendly progression
   • Auto-save prevents loss

7. Accessibility First
   • Keyboard navigation throughout
   • Screen reader compatible
   • High color contrast
   • Clear focus states
   • Semantic HTML structure
